0. はじめに（AI初心者の方へ）

Claude Code って何？

ひとことで言うと「日本語で指示するとパソコンが勝手に動くツール」 です。

  • メールの下書き、議事録の整理、データ集計、画像作り、HPの作成、ブラウザ操作…
  • 全部、日本語で「○○して」と頼むだけ で、PC上で自動的に作業してくれます。
  • ChatGPTとの違い：ChatGPTは「チャット画面の中だけ」。Claude Codeは「実際にPCを動かす」。

Claude Code でできること（具体例）

シーン あなたの指示 Claude Code がやること
営業 「この100社のHP見て、業種をまとめて」 100サイト全部開いて → 内容読み取り → 一覧表に出力
集計 「先月の売上をエクセルで集計して」 エクセル開く → 集計 → グラフ化 → 保存
制作 「整体院のLPを作って」 業界調査 → デザイン提案 → HTML生成 → プレビュー
リサーチ 「競合5社の料金プランを比較して」 各社サイト訪問 → 情報抽出 → 比較表作成
メール 「クライアント10社にお礼メール送って」 メール文生成 → Gmail起動 → 順番に送信（送信前確認あり）

何が嬉しいの？（メリット）

✅ 学習で賢くなる 教えたことを覚えて、次回から同じ説明不要

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

完了後、もう一度 brew install node

Windows:

winget install OpenJS.NodeJS.LTS

winget が使えない場合 → nodejs.org からLTS版をダウンロードしてインストール。
インストール後、Cursorを一度閉じて開き直してください。

確認: node --version で数字が出ればOK。

仕込み1: Plan Mode を使う（設計→実行）

これは何？

いきなり作業せず、Claude に 「こうやります」という設計図 を先に出させて、承認後に実行するモード。

例え話

美容師が「じゃ切りますね」とハサミを入れる前に、「こう切ります、長さこのくらい、前髪こうします、これでいいですか？」と先に見せてくれる感覚。

Before / After

Before（Plan Modeなし）:

あなた: 「整体院のLP作って」
Claude: → いきなり書き始める
（30分後）
Claude: 「できました！」
あなた: 「あれ、レビュー要素入ってない…」
Claude: 「やり直します」（追加30分）

After（Plan Mode使用）:

あなた: 「整体院のLP作って」（Plan Mode ON）
Claude: 「Plan: ①レビュー収集 ②強み抽出 ③HTML生成 ④検証
         画像はAI？写真ご支給？」
あなた: 「AIで。あとレビュー10件以上で」
Claude: 「OK 着手します」
（30分後）
Claude: 「完成、自己検証OK」

→ やり直し時間ゼロ

仕込み2: CLAUDE.md でプロジェクト学習させる

これは何？

プロジェクトのルートに置く 「Claude専用のマニュアル」。Claude が毎回開いて読んでから仕事を始める。

例え話

新人スタッフに「これ毎日読んでから仕事してね」と渡す業務マニュアル。
スタッフが入れ替わっても、新人がいきなり戦力になる。

Before / After

Before（CLAUDE.mdなし）:

月曜: 「日本語で応答してね」
火曜: 「日本語で応答してね」
水曜: 「日本語で応答してね」
（毎回ゼロから説明）

After（CLAUDE.mdあり）:

CLAUDE.mdに「日本語で応答」と1回書く
→ 永遠にそのルールで動く

さらにミスしたら即追記 する運用が最強：

あなた: 「住所、勝手に作ったでしょ。CLAUDE.mdに『住所は実データのみ』追記して」
Claude: 「追記しました」
→ 二度と勝手に住所を作らない

メリット

  • ✅ 同じ説明を毎回しなくていい
  • ✅ 過去のミスを永久に消せる
  • ✅ プロジェクトの好み・ルール・前提が全部蓄積
  • ✅ 別のPCに移っても CLAUDE.md があれば即仕事できる

デメリット

  • ⚠️ 書きすぎると逆にルールが守られなくなる（200行以内推奨）
  • ⚠️ 矛盾するルールが混ざると Claude が迷う → 整理が必要
  • ⚠️ ファイルを共有するときは個人情報に注意

CLAUDE.md の書き方の型

プロジェクトのルートに CLAUDE.md を作成（Claude Code に「CLAUDE.md を作って」と頼めばOK）。

# プロジェクトのルール

## 言語スタイル
- 日本語で応答
- 結論ファースト

## このプロジェクトで大事なこと
- ターゲット顧客: 個人事業主
- 単価: 月¥7,980
- 営業先: HPなしの店舗

## 頻出ミス記録（更新日: 2026-04-28）
- ❌ 個人情報を勝手に推測して書く → ✅ 本人に確認してから書く
- ❌ テンプレを流用する → ✅ 案件ごとに独自デザインで作る
- ❌ ランダムな画像URLを使う → ✅ AI生成 or 確認済みURLのみ

## 必ず守ること
- ファイルを公開する前に「公開していい案件か」確認
- メール送信前にスクリーンショットで1回チェック

## 納品前チェックリスト
- [ ] 誤字脱字なし
- [ ] 個人情報・連絡先が正しい
- [ ] スマホ表示でも崩れない
- [ ] リンクがすべて生きている

運用ルール（最重要）

Claude がミスしたら → その場で「CLAUDE.md に追記して」と頼む。これだけで賢くなり続けます。

（Claudeのミス発覚直後）
あなた: 「それ間違い。CLAUDE.md に『○○のときは××する』と追記して」
Claude: 「追記しました」
→ 次回以降、二度と同じミスは起きない

これは何？

Claude が作業を終えたら、自分で答え合わせ してから「完成」と報告する仕組み。

パン屋がパンを焼いたあと、自分で味見・形・焦げチェック → ダメなら作り直し。

簡易版（誰でもできる）

CLAUDE.md に 「納品前チェックリスト」 を書いておく（仕込み2の例にあり）。

そして指示の最後に必ずこう書く:

このHPを作って。完成したらCLAUDE.mdの納品前チェックリストに沿って
自己検証して、全部 ✅ になったら完成と報告して。

→ Claude が自分でチェックしてから完成報告してくれる。

上級版（自動化）

.claude/hooks/ フォルダに自動チェックスクリプトを配置すると、Claude の作業ごとに自動実行される。
- 例: HTML作ったら住所妥当性チェック
- 例: 応募文書いたら個人情報推測検出
- 例: 公開前に「公開禁止クライアント」確認

→ 上級者向け。まずは簡易版から。
